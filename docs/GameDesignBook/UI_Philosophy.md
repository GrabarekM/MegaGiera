# UI Philosophy

UI jest częścią świata, a nie nowoczesnym programem nałożonym na grę. Okna powinny przywoływać fizyczne przedmioty: księgę, pergamin, mapę, skrzynię albo sakiewkę.

HUD pozostawia eksploracji możliwie dużo przestrzeni. Pokazuje tylko informacje potrzebne w aktualnym kontekście, a szczegóły przenosi do otwieranych okien.

Każde podstawowe okno posiada widoczny i konsekwentny keybind. Ten sam klawisz otwiera i zamyka przypisane okno, a `Escape` wraca o jeden poziom.

Czytelność ma pierwszeństwo przed ornamentem. Tekst, liczby, aktywne stany, ostrzeżenia i dostępne akcje muszą być zrozumiałe bez znajomości dekoracyjnego języka interfejsu.

Interfejs nie powinien udawać fizycznego obiektu kosztem wygody. Metafora świata określa formę, lecz zachowanie pozostaje przewidywalne: stabilny układ, jasny fokus, spójne przyciski, jednoznaczne stany wyłączone i brak ukrytych krytycznych akcji.

UI nie podejmuje decyzji domenowych. Prezentuje stan gry i przekazuje intencje do odpowiednich systemów.
