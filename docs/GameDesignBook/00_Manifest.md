# Game Design Book — Manifest

> **Dokument nadrzędny:** [`../CONSTITUTION.md`](../CONSTITUTION.md). W przypadku sprzeczności Constitution ma pierwszeństwo przed Game Design Book, dokumentacją techniczną i aktualną implementacją. Zmiana Constitution wymaga jawnej decyzji właściciela projektu.

Centralny indeks decyzji projektowych MegaGiery. Dokumentacja opisuje intencję produktu; kod i testy opisują aktualną implementację. Rozbieżności należy rejestrować w `Repository_Analysis.md`, a nie ukrywać przez zmianę dokumentacji bez decyzji projektowej.

## Struktura

- `01_Design_Principles.md` — zasady podejmowania decyzji.
- `02_Player_Journey.md` — pełna podróż gracza.
- `03_Adventure_Structure.md` — budowa proceduralnej przygody.
- `04_Risk_vs_Reward.md` — koszt, ryzyko i nagroda.
- `05_Pillars.md` — filary doświadczenia.
- `06_Core_Gameplay_Rules.md` — niezmienne reguły rozgrywki.
- `07_UI_Philosophy.md` — miejsce UI w świecie.
- `08_Creative_Direction.md` — kierunek artystyczny.
- `09_Technical_Rules.md` — reguły techniczne.
- `10_Out_of_Scope.md` — świadome granice projektu.

Pierwsze rozwinięcia znajdują się w dokumentach bez prefiksu numerycznego. Numerowane rozdziały pozostają kontrolowanymi szkieletami do dalszego uzupełniania.

## Przyszłe rozdziały

- Słownik pojęć i nazewnictwa.
- Status zatwierdzenia poszczególnych rozdziałów.
- Historia istotnych rewizji Game Design Book.
